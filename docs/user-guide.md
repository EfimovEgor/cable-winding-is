# Руководство пользователя ИС намотки кабеля

## Как другим пользователям получить доступ

### В локальной сети

1. **На компьютере, где запущена ИС:**
   - Запустите API: `cd backend/api/CableWinding.Api` → `dotnet run`
   - Запустите TraceConnector: `cd backend/trace-connector/...` → `dotnet run`
   - Узнайте свой IP: `ipconfig` (строка IPv4-адрес, например `192.168.1.100`)

2. **Разрешите порт в брандмауэре Windows (если блокирует):**
   - Панель управления → Брандмауэр Windows → Дополнительные параметры
   - Правила для входящих подключений → Создать правило → Порт → TCP 5203

3. **Другие пользователи в сети открывают в браузере:**
   ```
   http://ВАШ_IP:5203
   ```
   Например: `http://192.168.1.100:5203`

### Через Docker

```powershell
cd cable-winding-is
docker-compose up -d
```
- Доступ: `http://localhost:5203` (локально) или `http://IP_СЕРВЕРА:5203` (из сети)

## Роли и возможности (по use-case диаграмме)

| Роль | Доступные разделы |
|------|-------------------|
| **Оператор АСУТП** | Мониторинг, Процесс, Инциденты, Отчёты, Архив, Возможности по ролям, API |
| **Технолог** | Нормативы, Архив, Возможности по ролям, API |
| **Сервисный инженер** | Диагностика, Возможности по ролям, API |

Каждая роль видит только свои разделы. При попытке открыть недоступный раздел — «Доступ запрещён».

---

## Текущий функционал (MVP)

- **Мониторинг** — просмотр текущих параметров намотки (скорость, натяжение, шаг)
- **API** — Swagger на `/swagger`
- **TraceConnector** — симуляция телеметрии (отправка каждые 2 с)

## Быстрый старт (для новых пользователей)

1. Откройте браузер, введите адрес: `http://IP_СЕРВЕРА:5203` (локально — `http://localhost:5203`)
2. На странице входа введите логин и пароль (см. таблицу выше)
3. После входа откроется страница «Мониторинг» с параметрами (если работает TraceConnector)
